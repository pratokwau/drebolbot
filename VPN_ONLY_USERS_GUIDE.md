# VPN-Only Users Access System

## Описание
Система двухуровневой авторизации позволяет предоставить пользователям с VPN ограниченный доступ к боту только на следующие команды:
- `/start` — главное меню
- `/status` — просмотр статуса бота и VPN
- `/myvpn` — управление собственным VPN
- `/settings` — настройки уведомлений (только перезагрузка бота и рассылки)

## Управление VPN-only пользователями

### Администратору в Python коде

```python
from handlers.utils import (
    add_vpn_only_user, 
    remove_vpn_only_user, 
    is_vpn_only_user,
    load_vpn_only_users
)

# Добавить пользователя в VPN-only список
add_vpn_only_user(user_id)

# Удалить пользователя из VPN-only списка (даёт полный доступ)
remove_vpn_only_user(user_id)

# Проверить, является ли пользователь VPN-only
if is_vpn_only_user(user_id):
    print("Пользователь имеет только доступ к VPN")

# Загрузить всех VPN-only пользователей
all_vpn_users = load_vpn_only_users()
```

### Хранилище данных
Данные хранятся в файле: `data/vpn_only_users.json`

Формат:
```json
[
  5185332965,
  8346942276
]
```

## Как это работает

### 1. Start Handler (`handlers/start.py`)
Уже поддерживает VPN-only пользователей:
- Если пользователь не авторизован И имеет привязку к VPN → показывает специальное меню с командами /myvpn, /settings, /help
- Если авторизован → показывает полное меню

### 2. Middleware (`middlewares/command_restriction.py`)
Проверяет команды для VPN-only пользователей:
- Если это VPN-only пользователь → проверяет доступ только к разрешённым командам
- Если команда не разрешена → отказывает с сообщением о доступных командах

### 3. Status Handler (`handlers/status.py`)
Позволяет VPN-only пользователям просматривать статус VPN и бота

### 4. Settings Handler (`handlers/settings.py`)
VPN-only пользователи видят упрощённое меню только с:
- Перезагрузка бота (вкл/выкл)
- Рассылки (вкл/выкл)

Им недоступны:
- Ежедневный отчёт
- Настройка времени отчёта
- Выбор голоса ИИ

### 5. MyVPN Handler (`handlers/xui/myvpn.py`)
Работает для всех пользователей с привязкой к VPN, независимо от авторизации

## Список разрешённых команд для VPN-only
Определены в `handlers/utils.py`:
```python
VPN_ONLY_ALLOWED_COMMANDS = ["/start", "/status", "/myvpn", "/settings"]
```

При попытке использовать другую команду пользователь получит сообщение:
```
⛔ У вас нет доступа к команде /command

Доступные команды: /start /status /myvpn /settings
```

## Примеры использования

### Сценарий 1: Новый пользователь с VPN, без полного доступа
1. Пользователь напишет в бот `/start`
2. Получит сообщение с информацией о привязанном VPN и доступных командах
3. Может использовать только `/start /status /myvpn /settings`

### Сценарий 2: Предоставить полный доступ VPN-пользователю
Администратор:
1. Добавляет пользователя в `authorized.json` (стандартная авторизация)
2. Удаляет из VPN-only списка: `remove_vpn_only_user(user_id)`
3. Теперь пользователь имеет полный доступ ко всем командам

### Сценарий 3: Ограничить доступ авторизованного пользователя только на VPN
Администратор:
1. Удаляет пользователя из `authorized.json`
2. Добавляет в VPN-only: `add_vpn_only_user(user_id)`
3. Теперь пользователь может использовать только разрешённые команды

## Файлы изменений
- `handlers/utils.py` — добавлены функции для управления VPN-only пользователями
- `middlewares/command_restriction.py` — обновлена логика проверки команд
- `handlers/status.py` — добавлена проверка доступа для VPN-only пользователей
- `handlers/settings.py` — добавлена поддержка упрощённого меню для VPN-only пользователей
- `data/vpn_only_users.json` — новый файл для хранения списка VPN-only пользователей (создаётся автоматически)
