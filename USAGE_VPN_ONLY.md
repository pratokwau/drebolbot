# 🔐 Система VPN-Only доступа

## Быстрый старт

### Что это?
Система для предоставления ограниченного доступа к боту пользователям, у которых есть привязанный VPN-сервис, но нет полного доступа к остальным функциям бота.

### Что видят VPN-only пользователи?
Только эти команды:
- `/start` — главное меню с информацией о VPN
- `/status` — просмотр статуса бота и VPN-сервиса
- `/myvpn` — управление своим VPN (добавить/удалить устройства)
- `/settings` — настройки (только перезагрузка бота и рассылки)

## Управление

### Через утилиту администратора

```bash
# Добавить пользователя в VPN-only (ограничить доступ)
python3 admin_vpn_only.py add 12345678

# Удалить из VPN-only (дать полный доступ, если в authorized.json)
python3 admin_vpn_only.py remove 12345678

# Список всех VPN-only пользователей
python3 admin_vpn_only.py list

# Проверить статус пользователя
python3 admin_vpn_only.py check 12345678
```

### Программно в Python коде

```python
from handlers.utils import (
    add_vpn_only_user,
    remove_vpn_only_user,
    is_vpn_only_user,
    load_vpn_only_users
)

# Добавить
add_vpn_only_user(user_id)

# Удалить
remove_vpn_only_user(user_id)

# Проверить
if is_vpn_only_user(user_id):
    print("Пользователь имеет VPN-only доступ")

# Список
all_vpn_users = load_vpn_only_users()
```

## Примеры использования

### Сценарий 1: Новый пользователь хочет только VPN

1. Пользователь пишет `/start`
2. Получает сообщение:
   ```
   🔐 drebol 🪼
   
   У вас подключён VPN-сервис.
   
   📱 Доступные команды:
   • /myvpn — Управление вашим VPN
   • /settings — Настройки уведомлений
   • /help — Связаться с администратором
   ```
3. Может пользоваться только этими командами

### Сценарий 2: Дать VPN-пользователю полный доступ

```bash
# Добавить в основной список авторизованных
echo "12345678" >> authorized.json  # или через админ-панель

# Удалить из VPN-only (если там был)
python3 admin_vpn_only.py remove 12345678
```

Теперь пользователь имеет полный доступ ко всем командам.

### Сценарий 3: Ограничить авторизованного пользователя

```bash
# Удалить из авторизованных
# (отредактировать authorized.json)

# Добавить в VPN-only
python3 admin_vpn_only.py add 12345678
```

Теперь доступ только через VPN.

## Как это работает внутри

1. **Middleware** (`middlewares/command_restriction.py`)
   - Проверяет каждую команду перед обработкой
   - Если VPN-only пользователь → проверяет разрешённые команды
   - Если запрещена → отказывает

2. **Status** (`handlers/status.py`)
   - VPN-only пользователи могут просматривать статус

3. **Settings** (`handlers/settings.py`)
   - VPN-only пользователи видят упрощённое меню
   - Только переключение perезагрузки и рассылок

4. **MyVPN** (`handlers/xui/myvpn.py`)
   - Работает для всех с привязанным VPN

## Файлы данных

- `data/vpn_only_users.json` — список ID пользователей с VPN-only доступом
- `authorized.json` — список всех авторизованных пользователей (обычный доступ)

## Приоритеты доступа

Если пользователь в обоих списках (авторизован И в VPN-only):
- **Приоритет: VPN-only** (ограниченный доступ побеждает)

Рекомендуется держать пользователей либо в одном, либо в другом списке.

## Ошибки и решения

### Пользователь видит: "⛔ У вас нет доступа к команде /rassstart"
**Решение:** Это нормально для VPN-only пользователей. Они могут использовать только: `/start /status /myvpn /settings`

### Пользователь не может менять все настройки в /settings
**Решение:** VPN-only пользователям доступны только:
- Перезагрузка бота
- Рассылки

### Как добавить новую разрешённую команду для VPN-only?
Отредактируйте в `handlers/utils.py`:
```python
VPN_ONLY_ALLOWED_COMMANDS = ["/start", "/status", "/myvpn", "/settings", "/help"]
```

## Автоматизация

Можно создать скрипт для автоматического добавления пользователей:

```python
#!/usr/bin/env python3
from handlers.utils import add_vpn_only_user

vpn_user_ids = [123456, 789012, 345678]

for uid in vpn_user_ids:
    add_vpn_only_user(uid)
    print(f"Добавлен: {uid}")
```
